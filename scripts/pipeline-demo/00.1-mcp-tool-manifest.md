# 输入④：MCP 工具清单（LLM 编排时需要的工具知识）

## 页面级工具（不需 ref，全页面搜索，最终翻译为裸 expect）

### browser_navigate
- 参数: `url: string`
- 说明: 导航到指定 URL

### browser_snapshot
- 参数: 无
- 说明: 获取当前页面的 ARIA 可访问性树快照，包含所有元素的 role、name、ref
- 用途: LLM 从中提取目标元素的 ref 值

### browser_verify_text_visible
- 参数: `text: string`
- 说明: 验证文本在页面上可见（全页面搜索，匹配包含该文本的任意元素）
- 输出: 裸 `await expect(...).toBeVisible()`
- 示例: `{text: "设备管理"}` → `await expect(page.getByText('📱 设备管理平台')).toBeVisible();`

### browser_verify_text_not_visible
- 参数: `text: string`
- 说明: 验证文本在页面上**不**可见
- 输出: 裸 `await expect(...).toHaveCount(0)`

### browser_verify_page_url
- 参数: `expected: string`, `mode: "equals" | "contains"`（可选，默认 equals）
- 说明: 验证当前页面 URL
- 输出: 裸 `await expect(page).toHaveURL(...)`

## 元素级工具（需要 ref/selector，最终翻译为 robustLocate）

### browser_verify_state
- 参数:
  - `element: string` — 人类可读的元素描述
  - `ref: string` — 从快照提取的元素引用
  - `selector: string`（可选）— CSS 或 role 选择器
  - `state: "disabled" | "enabled" | "editable" | "focused" | "empty" | "hidden" | "inDOM" | "attached" | "hasClass" | "visible"` — 期望状态
  - `classExpected: string`（可选）— state=hasClass 时必填，空格分隔的 CSS class
- 说明: 验证元素的交互状态
- 输出: `await expect(locator).toBe{State}()`
- 示例: `{element:"Apple标签", ref:"e23", state:"visible"}` → `await expect(page.locator('#filter-panel').getByText('Apple')).toBeVisible();`

### browser_verify_text
- 参数:
  - `element: string`, `ref: string`, `selector: string`（可选）
  - `text: string` — 期望的文本值
  - `mode: "equals" | "contains" | "regex"`（可选，默认 equals）
- 说明: 验证元素文本内容（含空白符归一化）
- 输出: `await expect(locator).toHaveText(...)` / `toContainText(...)`

### browser_verify_attribute
- 参数:
  - `element: string`, `ref: string`, `selector: string`（可选）
  - `name: string` — 属性名（如 `href`, `data-status`, `aria-expanded`）
  - `expected: string` — 期望值
- 说明: 验证元素属性值
- 输出: `await expect(locator).toHaveAttribute(name, expected)`

### browser_verify_css
- 参数:
  - `element: string`, `ref: string`, `selector: string`（可选）
  - `property: string` — CSS 属性名（如 `color`, `font-size`, `display`）
  - `expected: string` — 期望的 computed value
- 说明: 验证元素的 computed CSS 属性值
- 内部: 使用 `window.getComputedStyle(el).getPropertyValue(prop)` 获取实际值
- 输出: `await expect(locator).toHaveCSS(property, expected)`

### browser_verify_count
- 参数:
  - `text: string`（页面级）**或** `ref: string` + `element: string`（元素级）
  - `selector: string`（可选）
  - `count: number` — 期望数量
  - `operator: "equals" | "greater_than" | "less_than" | "greater_than_or_equal" | "less_than_or_equal"`（可选，默认 equals）
- 说明: 验证匹配元素的数量
- 页面级（仅 text）→ 裸 expect; 元素级（有 ref/selector）→ robustLocate

### browser_verify_element_visible
- 参数: `role: string`, `accessibleName: string`
- 说明: 通过 ARIA role + accessible name 验证元素可见
- 输出: `await expect(locator).toBeVisible()`

### browser_verify_value
- 参数:
  - `type: "textbox" | "checkbox" | "radio" | "combobox" | "slider"`
  - `element: string`, `ref: string`, `selector: string`（可选）
  - `value: string` — checkbox 用 "true"/"false"
- 说明: 验证表单元素的值/选中状态

### browser_verify_list_visible
- 参数:
  - `element: string`, `ref: string`, `selector: string`（可选）
  - `items: string[]` — 期望的列表项文本数组
- 说明: 验证列表包含指定项，用 `toMatchAriaSnapshot` 匹配

### browser_click
- 参数: `element: string`, `ref: string`, `selector: string`（可选）
- 说明: 点击元素
- 输出: `await locator.click()`

## LLM 编排时如何使用本清单

给定自然语言用例：

1. **语义匹配**: "验证 X 可见" → `verify_text_visible`（页面级）或 `verify_state(state=visible)`（元素级）
2. **判断页面级/元素级**: 涉及"页面上任意位置" → 页面级；涉及"某个具体按钮/标签" → 元素级，需要 ref
3. **提取 ref**: 从 snapshot 中搜索目标的 ARIA 条目，取 `[ref=eXX]`
4. **组装参数**: element（人类描述）+ ref（技术定位）+ state/property/expected（验证内容）
