# 阶段 1.5：LLM 工具编排（自然语言 → MCP 工具调用序列）

## 输入
- 自然语言用例（01-input-natural-language.md）
- 页面快照（ARIA 树，浏览器自动生成）

## LLM 推理过程

### Step 1：解析用例步骤

```
用例原文：
  前置条件：筛选面板品牌标签可见、表格有 30 条数据、分页器可见
  执行步骤：点击筛选面板中的 Apple 品牌标签
  预期结果：表格只显示 Apple 设备、计数更新、Apple 标签保持可见
```

LLM 拆解为：
```
前置验证：
  A. 确认当前页面是 device-manager → browser_verify_page_url
  B. 确认"设备管理"标题可见 → browser_verify_text_visible
  C. 确认筛选面板存在 → browser_verify_text_visible("品牌") + browser_verify_text_visible("操作系统")
  D. 确认计数为 30 → browser_verify_text_visible("30")
  E. 确认品牌标签可见 → browser_verify_state(ref=品牌ref, state=visible)
  F. 确认表格可见 → browser_verify_state(ref=表格ref, state=visible)

执行：
  G. 点击 Apple 标签 → browser_click(ref=Apple ref)

预期结果：
  H. Apple 标签仍然可见 → browser_verify_state(ref=Apple ref, state=visible)
  I. 表格内容变化（可选）→ 读快照对比
```

### Step 2：从快照提取 ref

LLM 读到以下 ARIA 快照片段：

```yaml
- generic [ref=e20]:
  - generic [ref=e21]: 品牌
  - generic [ref=e22]:
    - generic [ref=e23] [cursor=pointer]: Apple      ← 这是要点击的
    - generic [ref=e24] [cursor=pointer]: Samsung
    - generic [ref=e25] [cursor=pointer]: 华为
- heading "设备筛选 30" [ref=e17]                      ← 计数在这里
- table [ref=e89]:                                     ← 表格在这里
  - rowgroup [ref=e90]:
    - row "设备ID ▲ 品牌 型号..." [ref=e91]
```

LLM 推理：
```
ref=e23 → Apple 筛选标签 → 用于 browser_verify_state + browser_click
ref=e24 → Samsung 标签 → 用于 browser_verify_state
ref=e25 → 华为标签 → 用于 browser_verify_state
ref=e89 → 设备表格 → 用于 browser_verify_state
ref=e17 → 包含 "30" → 不需要 ref，直接用 browser_verify_text_visible
```

### Step 3：生成 MCP 调用序列

```json
[
  {"tool": "browser_verify_page_url",    "args": {"expected": "device-manager", "mode": "contains"}},
  {"tool": "browser_verify_text_visible", "args": {"text": "设备管理"}},
  {"tool": "browser_verify_text_visible", "args": {"text": "品牌"}},
  {"tool": "browser_verify_text_visible", "args": {"text": "操作系统"}},
  {"tool": "browser_verify_text_visible", "args": {"text": "30"}},
  {"tool": "browser_verify_state",        "args": {"element": "Apple品牌标签",  "ref": "e23", "state": "visible"}},
  {"tool": "browser_verify_state",        "args": {"element": "Samsung品牌标签", "ref": "e24", "state": "visible"}},
  {"tool": "browser_verify_state",        "args": {"element": "华为品牌标签",   "ref": "e25", "state": "visible"}},
  {"tool": "browser_verify_state",        "args": {"element": "设备表格",       "ref": "e89", "state": "visible"}},
  {"tool": "browser_click",              "args": {"element": "Apple品牌筛选标签", "ref": "e23"}},
  {"tool": "browser_verify_text_visible", "args": {"text": "Apple"}}
]
```

## 说明
这一步目前由 `run-test-case.cjs` 硬编码执行。在真实系统中，LLM 读取自然语言用例和快照后，自动生成上述工具调用序列，再逐条调用 MCP 工具。

关键在于 LLM 需要理解：
1. **语义 → 工具映射**："验证 X 可见" → browser_verify_text_visible 或 browser_verify_state
2. **快照 → ref 提取**：从 ARIA 树中找到目标元素，拿到 ref 值
3. **参数组装**：把 ref + element 描述 + state 拼成完整的调用参数
