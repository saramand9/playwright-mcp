# 输入③：被测页面结构（device-manager.html）

## 页面信息
- URL: `http://localhost:8080/device-manager.html`
- 标题: 设备管理平台 — Playwright 测试靶场
- 功能: 设备列表 + 品牌/操作系统/内存等多维筛选

## 关键 DOM 结构（从 ARIA 快照还原）

### 导航栏
```
navigation [ref=e5]
  link "🏠 首页" → index.html
  link "🧪 综合表单" → test-site.html
  ...
  link "📱 设备管理" → device-manager.html
```

### 筛选面板
```
#filter-panel
  heading "设备筛选 30"          ← "30" 是设备总数
  品牌:
    Apple   [ref=e23, cursor=pointer]   ← 可点击的筛选标签
    Samsung [ref=e24, cursor=pointer]
    华为    [ref=e25, cursor=pointer]
    小米    [ref=e26, cursor=pointer]
    OPPO    [ref=e27, cursor=pointer]
    Vivo    [ref=e28, cursor=pointer]
    一加    [ref=e29, cursor=pointer]
  操作系统:
    iOS         [ref=e33, cursor=pointer]
    Android     [ref=e34, cursor=pointer]
    HarmonyOS   [ref=e35, cursor=pointer]
  运行内存: ...
  存储容量: ...
```

### 设备表格
```
table [ref=e89]
  表头: 设备ID ▲ 品牌 型号 操作系统 系统版本 屏幕尺寸 RAM 存储 电池 网络 价格 状态 采购日期 部门 NFC 刷新率
  数据行: 30 行，每行包含 DEV-xxx 的设备信息
```

## 页面交互逻辑
- 点击品牌标签 → 筛选面板标签高亮，但**需额外点击"查询"按钮**才触发表格过滤
- 当前测试中点击 Apple 标签后表格未过滤（计数未变），因为未触发查询动作
